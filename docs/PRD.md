# PRD — Win11 Right-Edge Calendar (WPF/.NET 8)

## 1. 背景
Windows標準の通知/カレンダーでは機能が不足。
右端ホバーで自然に出る常駐パネルとして、素早く予定を確認し、その場で追加/編集したい。
軽量性を最優先とし、WebView2/Electronの同梱的アプローチは採らない。

## 2. 目標
- 右端ホバーで即座に表示される「パネル型カレンダー」
- Googleカレンダーと同期し、パネル内でCRUDが完結
- 常駐しても軽い

## 3. 非目標（MVP外）
- 参加者/招待/RSVP
- 繰り返し予定の完全対応（RRULE、例外）
- 通知/リマインダー（将来対応の拡張性は確保する）

## 4. 主要要件（Must）
### 4.1 右端表示
- 右端2〜6pxに侵入し100ms滞在で表示
- フェードイン/アウト
- NoActivate
- マウスアウト猶予250ms
- マルチモニタ対応（カーソルがいるモニタ）

### 4.2 フルスクリーン抑制
- 前面がフルスクリーンならホバー表示しない
- ホットキーは表示を許可（設定で抑制余地）

### 4.3 同期
- 単一アカウント
- 複数カレンダー統合表示、カレンダー単位ON/OFF
- 同期窓は「過去31日〜未来31日」で固定（MVP）
- CRUD（追加/編集/削除）
- 終日予定対応、タイムゾーン整合

### 4.4 競合
- サーバー優先
- 競合発生時は上書き禁止
- ローカル編集案は退避ログ化

## 5. UX（MVP）
- 月カレンダー + 選択日リスト
- 追加/編集はフォーム（タイトル必須、開始/終了、終日、場所、メモ、追加先カレンダー）

## 6. 成功指標（目安）
- dwell成立後、体感即時に出る
- 常駐アイドル時のCPU/メモリが増えにくい
- 予定追加が数クリックで終わる

## 7. 将来拡張の方針
- 通知はEventsテーブルに無理やり埋めず、別テーブル（NotificationRules）で足す
- 同期窓は将来「閲覧月に追随してスライド」可能な設計にする
